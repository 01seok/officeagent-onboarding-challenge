### Problem (나빴던 것)

- **Retrobot 반복 실패 (4회 모두)**: `retros/` 디렉토리 미생성, Write 권한 거부, Read 토큰 한도 초과가 반복되었다. 같은 원인으로 4개 세션 모두 실패 — 실질적 회고 결과물이 없었다.
- **JSONL 로그 파일 크기 병목**: 큰 로그 파일 전체 Read 시도 → 토큰 초과 → Grep 대안 시도 → 출력 초과로 이어지는 비효율 패턴이 여러 세션에서 반복되었다.
- **중복 실행 감지 없음**: 동일 커밋에 대해 Retrobot이 여러 차례 실행되었지만 이미 생성된 회고를 감지하는 로직이 없어 중복 시도가 반복되었다.
- **개인 작업 문서 초기 미관리**: `ver_claude.md`, `officeagent_challenge_approach.md` 같은 개인 파일이 초기에 gitignore에서 빠져 뒤늦게 원격에서 제거 작업이 필요했다.

### Try (개선해야 할 것)

- **`retros/` 디렉토리를 프로젝트 초기화 시점에 생성**: `.gitkeep` 파일과 함께 디렉토리를 미리 커밋해두어 Write 권한 문제를 원천 차단한다.
